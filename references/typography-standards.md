# 中文技术书排版参数标准参考

> 基于出版行业标准、O'Reilly/Manning 等头部技术出版商实践、以及中文排印学通用规范整理。用于指导 `layout.md` 和 `booksmith-typst.py` 的间距配置，避免凭感觉随意填数。

## 核心原则

1. **段间距 ≥ 行间距，但不宜过大** — 知乎版式设计
2. **标题下方间距随级别递减** — H3 下方应明显小于 H2 下方
3. **列表项间距 ≤ 段落间距** — 列表已有 bullet/number 作为视觉分隔
4. **行距 120%~180% 适合阅读** — 网格系统构建方法论文
5. **技术书正文 10~12pt，行距 1.5~1.8 倍** — 中文出版通用

## 推荐参数表（modern 风格 / A4 / 11pt 正文）

| 参数 | 推荐值 | 取值依据 |
|------|--------|---------|
| **正文字号** | 10.5~11pt | O'Reilly 9.5pt（书宋），中文技术书需略大 |
| **行内间距 (leading)** | **1.6~1.7em** | 中文书籍 1.5~1.8 倍；技术书偏紧凑但 ≥1.5 |
| **段落间距** | **1.0~1.2em** | 无缩进现代排版用段间距替代缩进；≥ 行距的 0.6 倍 |
| **H1 上方** | 2.0~2.2em | 每章新页，需明显留白 |
| **H1 下方** | 1.2~1.3em | 教材/论文通用 |
| **H2 上方** | 1.5~1.8em | 节标题与正文之间 |
| **H2 下方** | **0.8~1.0em** | 应小于 H1 下方，保持层级递减 |
| **H3 上方** | 1.2~1.3em | 子节标题 |
| **H3 下方** | **0.5~0.6em** | 明显小于 H2 下方，子节与正文更紧密 |
| **列表项间距** | **0.5~0.8em** | 列表已有 bullet 分隔，间距应 < 段落间距 |
| **代码块内边距** | 8~10pt | O'Reilly 实践 |
| **页边距（上/右/下/左）** | 22/20/28/20 mm | 技术书通用 |

## 常见错误配置

| 错误 | 问题 | 修正 |
|------|------|------|
| leading = 1.3em | 接近单倍行距，中文长段落阅读疲劳 | 提升到 1.6~1.7em |
| paragraph_spacing = 1.5em | 段落间"断层"感，页面松散 | 降到 1.0~1.2em |
| H2 下方 = H3 下方 | 层级扁平化，读者分不清 H2/H3 | H3 下方应明显更小 |
| list_spacing = 1.3em | 列表项像独立段落，失去紧凑感 | 降到 0.5~0.8em |
| H1 下方 < H2 上方 | 章节过渡时视觉跳跃 | 保持 H1 下方 ≥ H2 下方 |

## layout.md 覆盖语法

在 `layout.md` 中可以用自然语言覆盖预设参数（booksmith-typst.py 自动解析）：

```yaml
# 基础覆盖（已支持）
正文行高：1.65
正文字号：10.5pt
margin: 2.5cm 2.2cm 2.5cm 2.2cm

# 标题间距覆盖（v1.1.1+ 支持）
H1 上方间距：2.0
H1 下方间距：1.2
H2 上方间距：1.5
H2 下方间距：0.9
H3 上方间距：1.2
H3 下方间距：0.6

# 列表项间距覆盖（v1.1.1+ 支持）
列表项间距：0.7
```

## 风格差异速查

| 风格 | 正文字体 | 行距 | 段距 | 首行缩进 | 适用 |
|------|---------|------|------|---------|------|
| classic（经典） | Noto Serif SC | 1.8em | 1.2em | 2em | 文学、人文 |
| **modern（技术书）** | Noto Sans SC | **1.6~1.7em** | **1.0~1.2em** | 无 | **技术、科普、商业** |
| academic（学术） | Noto Serif SC | 1.6em | 0.8em | 1.5em | 论文、研究报告 |
| minimal（极简） | Noto Sans SC | 1.9em | 1.5em | 无 | 设计、画册 |
